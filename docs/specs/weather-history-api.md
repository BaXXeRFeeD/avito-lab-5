# Weather History API

## Контекст
Фича представляет
собой небольшой HTTP API, который возвращает исторические погодные данные за
выбранный диапазон дат, запрашивая архив у Open-Meteo.

## Что строим
- HTTP endpoint `GET /weather-history`
- Query-параметры:
  - `latitude`
  - `longitude`
  - `start_date`
  - `end_date`
- Сервисный слой, который обращается к Open-Meteo за архивной погодой
- Нормализованный JSON-ответ, где каждый день представлен отдельным объектом

## Что не строим
- Не добавляем базу данных и постоянное хранилище
- Не добавляем авторизацию
- Не добавляем поиск по названию города, поддерживаются только координаты
- Не добавляем кэширование и rate limiting
- Не добавляем frontend

## Критерии приемки
1. Запрос с корректными координатами и корректным диапазоном дат возвращает HTTP 200
2. Ответ содержит поля:
   - `location`
   - `range`
   - `days`
   - `source`
3. Каждый объект в `days` содержит поля:
   - `date`
   - `temperature_max`
   - `temperature_min`
   - `precipitation_sum`
4. Если `start_date` больше `end_date`, API возвращает HTTP 400 и понятное сообщение об ошибке
5. Если внешний погодный сервис возвращает некорректный payload, API возвращает HTTP 502 и понятное сообщение об ошибке

## Граничные случаи
- `start_date == end_date`
- Координаты на допустимых числовых границах
- Внешний сервис вернул пустой блок `daily`
- Внешний сервис вернул массивы разной длины внутри `daily`
